Redrift Lane

Rebuild without losing graph discipline.

Redrift creates phase tasks, copies relevant fences, executes suite checks per phase, and keeps all decisions visible in Workgraph.

Principles

Fast, supervised-by-design, and loop-safe.

No hard blocks by default

Findings become logs and deterministic follow-up tasks, not silent divergence or dead-stop workflows.

Model-mediated decisions

Contracts and fences encode intent where agents execute, reducing memory drift across sessions.

Bounded recursion

Cooldowns, action budgets, and circuit breakers prevent endless self-healing loops.

Starter Kit

How friends should use it on day one.

1. Install `driftdriver` + `coredrift`, add optional lanes as needed.

2. In app repo: `wg init` then `driftdriver install ...`

3. Start tasks with ./.workgraph/drifts check --task <task-id> --write-log --create-followups

4. For brownfield rebuilds, run `redrift wg execute` in a v2 workspace.
